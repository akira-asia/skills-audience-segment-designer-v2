# Meta広告 オーディエンスセグメント設定リファレンス
# audience-segment-designer v4.1 参照ファイル
# 更新日：2026-03-23

このファイルはSTEP4で参照する。STEP2-5で確定したセグメント×ターゲティング軸を、Meta広告（Facebook/Instagram）の管理画面で実際に設定可能なオーディエンス定義に変換する。

---

## 1. オーディエンスの種類と用途

| オーディエンス種別 | 目的 | ファネル位置 | 設定レベル | セグメント軸との対応 |
|:---------------|:----|:----------|:---------|:----------------|
| **コアオーディエンス** | Meta保有データで属性・興味関心・行動からリーチ | 全ファネル | 広告セット | デモグラフィック/ニーズ/購買行動/ライフイベント |
| **カスタムオーディエンス** | 自社データ（サイト/CRM/アプリ/エンゲージメント）からリーチ | 下位ファネル（リマーケティング） | 広告セット | 購買行動ベース |
| **類似オーディエンス** | カスタムオーディエンスに類似するユーザーにリーチ | 中位ファネル | 広告セット | 拡張リーチ |
| **Advantage+ オーディエンス** | MetaのAIが最適なオーディエンスを自動探索 | 全ファネル | 広告セット | AI最適化 |

## 2. コアオーディエンス（詳細ターゲット設定）

### 基本ターゲティング

| 設定項目 | 内容 | 設定パス |
|:--------|:----|:------|
| **地域** | 国/地域/都市/郵便番号/半径指定 | 広告セット > 地域 |
| **年齢** | 13〜65歳以上（1歳刻み） | 広告セット > 年齢 |
| **性別** | すべて/男性/女性 | 広告セット > 性別 |
| **言語** | 使用言語 | 広告セット > 言語 |
| **配置** | Facebook/Instagram/Audience Network/Messenger | 広告セット > 配置 |

### 詳細ターゲティング（flexible_spec）

詳細ターゲティングはOR/AND/除外の論理演算で組み合わせ可能。

**論理演算の仕組み：**
```
同一カテゴリ内の複数項目 → OR（いずれか）
異なるカテゴリ間 → AND（すべて）
除外 → NOT（対象外）

例：「興味関心：グルメ OR お取り寄せ」AND「行動：EC利用者」NOT「興味関心：ファストフード」
```

### A. デモグラフィック

| カテゴリ | サブカテゴリ | 設定例 | ビジネスモデル型との親和性 |
|:--------|:---------|:-----|:------------|
| **学歴** | 学歴レベル/専攻/学校名 | 大学卒以上 | E-1型 |
| **収入** | 世帯収入帯 | 上位25%/上位10% | D-1型、D-3型 |
| **ライフイベント** | 最近引越し/婚約中/新婚/最近転職 | 「最近引越し」→B-1型 | B-1型 |
| **子供の有無** | 子供の年齢別（0-2歳/3-5歳/6-8歳/8-12歳/13-18歳） | 「子供あり（0-2歳）」 | A-1型、B-1型 |
| **交際ステータス** | 独身/交際中/婚約/既婚/離婚 | 「既婚」 | D-3型（ギフト層） |
| **勤務先** | 企業名/業種/役職 | 「マーケティング担当」 | B-2型、C-1型 |
| **住宅** | 持ち家価格帯 | — | B-1型 |

**設定方法：**
```
管理画面パス：広告セット > オーディエンス > 詳細ターゲティング > 利用者層
入力：カテゴリから選択、または検索窓でキーワード検索
```


## 3. カスタムオーディエンス

広告アカウントあたり最大500個作成可能。

### A. ウェブサイトトラフィック（Metaピクセル）

| 設定項目 | 内容 |
|:--------|:----|
| ソース | Metaピクセル |
| ルール条件 | URL訪問/特定イベント（購入・カート追加等）/滞在時間 |
| 保持期間 | 1〜180日（ローリング） |
| 最小サイズ | 推奨1,000人以上（類似オーディエンス利用時） |

**設定方法：**
```
管理画面パス：オーディエンス > カスタムオーディエンス > ウェブサイト
設定項目：
  1. ピクセルを選択
  2. イベント選択（すべての訪問者/特定ページ訪問者/特定イベント）
  3. URL条件（含む/等しい/含まない）
  4. 保持期間（日数）
  5. 除外条件（任意）
  6. オーディエンス名

設定例（D-3型 ギフトEC）：
  名前：「過去90日以内のサイト訪問者（購入なし）」
  条件：すべてのウェブサイト訪問者（過去90日）
  除外：Purchase イベント発生者（過去90日）
```

### B. 顧客リスト（CRM）

| 設定項目 | 内容 |
|:--------|:----|
| アップロード形式 | CSV/TXT |
| 使用可能な識別子 | メール/電話番号/名前/都市/都道府県/郵便番号/国/生年月日/性別/モバイル広告ID |
| ハッシュ化 | SHA256必須（アップロード前にハッシュ化） |
| 最小レコード数 | 推奨1,000件以上 |

**設定方法：**
```
管理画面パス：オーディエンス > カスタムオーディエンス > 顧客リスト
設定項目：
  1. データソース選択（ユーザー提供のみ/パートナー提供含む）
  2. CSVファイルアップロード
  3. 識別子のマッピング（列名とデータ型の紐付け）
  4. オーディエンス名
正規化ルール：
  - メール：小文字化、前後の空白除去
  - 電話：国番号付き（例：+81XXXXXXXXXX）
  - 名前：小文字化、特殊文字除去
```

**離脱顧客セグメントへの活用例：**
```
セグメント①：忘却型×直近6ヶ月
  → CRMから「最終購入日6ヶ月以内 × 購入回数1回以上 × クレーム履歴なし」を抽出しアップロード

セグメント③：2〜3回購入後離脱
  → CRMから「購入回数2-3回 × 最終購入日13ヶ月以上前」を抽出しアップロード
```

### C. アプリアクティビティ

| 設定項目 | 内容 |
|:--------|:----|
| ソース | Facebook SDK/アプリイベント |
| イベント条件 | インストール/アプリ内購入/レベル達成/決済情報追加 |
| 保持期間 | 1〜180日 |

### D. エンゲージメント

| ソース | 対象アクション | 保持期間 |
|:------|:-----------|:------|
| **動画** | 3秒/10秒/15秒再生、25%/50%/75%/95%視聴 | 1〜365日 |
| **リードフォーム** | フォーム開封/送信 | 1〜90日 |
| **インスタントエクスペリエンス** | 開封/クリック | 1〜365日 |
| **Facebookページ** | ページ訪問/いいね/投稿エンゲージメント | 1〜365日 |
| **Instagramプロフィール** | プロフィール訪問/投稿エンゲージメント/広告エンゲージメント | 1〜365日 |
| **ショッピング** | 商品閲覧/カート追加 | 1〜365日 |

**設定方法：**
```
管理画面パス：オーディエンス > カスタムオーディエンス > Metaのソース > [ソース種別]
設定項目：
  1. ソース選択（動画/リードフォーム/ページ等）
  2. アクション条件選択
  3. 対象コンテンツ選択（特定の動画/ページ等）
  4. 保持期間
  5. オーディエンス名
```

## 4. 類似オーディエンス（Lookalike）

カスタムオーディエンスを「ソース」として、類似するユーザーを自動検出。

| 設定項目 | 内容 | 推奨値 |
|:--------|:----|:-----|
| **ソースオーディエンス** | カスタムオーディエンスを選択 | 1,000〜5,000人が最適 |
| **対象国** | 類似ユーザーを探す国 | 通常は日本（JP） |
| **類似度（%）** | 1%〜20%（1%刻み） | 1%＝最も類似（高精度）、10%以上＝リーチ優先 |
| **帯域指定** | 開始%〜終了%で特定帯域を指定可能 | 例：1%→2%（上位1%を除いた層） |

**設定方法：**
```
管理画面パス：オーディエンス > 類似オーディエンス > 作成
設定項目：
  1. ソースオーディエンス選択
  2. 対象地域選択
  3. 類似度%設定
  4. （任意）帯域指定
作成時間：6〜24時間
ソースあたり最大500個
```

**推奨の類似度設定：**

| 目的 | 類似度 | 理由 |
|:----|:------|:----|
| 高精度リマーケティング拡張 | 1% | ソースに最も近い。CVR重視 |
| バランス型拡張 | 1%〜3% | 精度とリーチの両立 |
| 認知拡大 | 5%〜10% | リーチ重視。CPAは上昇する |

**ソースオーディエンスの推奨：**

| 優先度 | ソース | 理由 |
|:------|:-----|:----|
| 最優先 | 購入者リスト（CRM） | 最も質が高い。LTV上位者に絞るとさらに効果的 |
| 高優先 | ウェブサイトCV者 | 直近のコンバージョン行動に基づく |
| 中優先 | ウェブサイト訪問者（上位25%滞在時間） | 関心度の高い訪問者 |
| 参考 | ページエンゲージメント | ファネル上位向け。精度は低め |

## 5. Advantage+ オーディエンス（AI最適化）

MetaのAIが最適なオーディエンスを自動探索する機能。従来の手動ターゲティングに代わる推奨アプローチ。

| 設定項目 | 内容 |
|:--------|:----|
| **オーディエンスサジェスション** | 従来のターゲティング設定を「推奨」として入力 |
| **動作** | Metaが推奨を起点に自動拡張。コンバージョン最適化 |
| **推奨入力** | カスタムオーディエンス/興味関心/デモグラフィック |

**設定方法：**
```
管理画面パス：広告セット > Advantage+ オーディエンス
設定項目：
  1. オーディエンスサジェスション（任意）
    - カスタムオーディエンス
    - 詳細ターゲティング（興味関心・行動・利用者層）
    - 年齢・性別・地域
  2. 除外設定（任意）
注意：サジェスションはあくまで推奨。Metaは設定外にも配信を拡張する
```

## 6. 特別広告カテゴリの制限

住宅・雇用・金融サービス関連の広告には以下の制限がある。

| 制限項目 | 内容 |
|:--------|:----|
| 年齢 | 18〜65歳以上のみ（絞り込み不可） |
| 性別 | すべての性別（絞り込み不可） |
| 地域 | 半径15マイル以上のみ。郵便番号指定不可 |
| 詳細ターゲティング | 学歴・収入・子供の有無・交際ステータス等が使用不可 |
| 類似オーディエンス | 使用不可 |

## 7. STEP4出力テンプレートへの適用ガイド

### セグメント軸 → Meta広告設定の変換ルール

| セグメント軸 | 第1候補の設定方法 | 第2候補（第1が使えない場合） |
|:----------|:-------------|:---------------------|
| デモグラフィック（属性） | 基本設定（年齢・性別・地域）＋詳細ターゲティング（利用者層） | — |
| 購買行動（行動） | カスタムオーディエンス（CRM/ピクセル）＋詳細ターゲティング（行動：購買行動） | 類似オーディエンス（購入者ソース） |
| 検討行動（状況） | カスタムオーディエンス（ピクセル：特定ページ訪問）＋詳細ターゲティング（興味関心） | ※Metaにはキーワードターゲティングなし。興味関心で代替 |
| 興味関心（ニーズ） | 詳細ターゲティング（興味・関心） | カスタムオーディエンス（エンゲージメント） |
| ライフイベント（時間） | 詳細ターゲティング（利用者層：ライフイベント） | — |

### 自社データの有無による設定パターン

| 自社データの状況 | 推奨設定パターン |
|:------------|:-----------|
| **CRMリストあり（1,000件以上）** | カスタムオーディエンス（顧客リスト）→ 類似1%〜3% が最優先 |
| **ピクセル設置済み・訪問者あり** | カスタムオーディエンス（ウェブサイト）→ 類似1%＋リマーケティング |
| **自社データなし** | コアオーディエンス（詳細ターゲティング）→ Advantage+で最適化 |

### Google広告との主な違い

| 項目 | Google広告 | Meta広告 |
|:----|:---------|:-------|
| 検索意図ターゲティング | カスタムセグメント（検索KW） | **なし**（興味関心で代替） |
| キーワード指定 | カスタムセグメント | **なし** |
| URL指定 | カスタムセグメント（類似サイト訪問者） | **なし** |
| 購買意向セグメント | 約26カテゴリの公式セグメント | 行動＋興味関心で類似構築 |
| 類似オーディエンス | デマンドジェネレーションのみ | **全キャンペーンで利用可能** |
| AI最適化 | P-MAXのオーディエンスシグナル | Advantage+ オーディエンス |
| ターゲティングモード | ターゲティング/モニタリング選択可 | ターゲティングのみ（常に制限的） |
| オーディエンスの論理演算 | 統合セグメント（AND/OR） | flexible_spec（OR内AND除外） |

**セグメント設計での実務上の注意：**
- Google広告の「検討行動（状況）」軸はカスタムセグメントで直接対応できるが、Meta広告には同等機能がない → **興味関心＋カスタムオーディエンス（ピクセル）で代替する**
- Meta広告の強みは**類似オーディエンス**。CRMデータがある場合、Googleよりも効率的にリーチ拡大できる
- **Advantage+** は十分なコンバージョンデータ（週50件以上推奨）がある場合に最も効果的
