# Google広告 オーディエンスセグメント設定リファレンス
# audience-segment-designer v3.5 参照ファイル
# 更新日：2026-03-23

このファイルはSTEP4で参照する。STEP2-5で確定したセグメント×ターゲティング軸を、Google広告の管理画面で実際に設定可能なオーディエンス定義に変換する。

---

## 1. オーディエンスセグメントの種類と用途

| セグメント種別 | 目的 | ファネル位置 | 対応キャンペーン | セグメント軸との対応 |
|:-------------|:----|:----------|:-------------|:----------------|
| **アフィニティセグメント** | 長期的な興味関心・ライフスタイルに基づくリーチ | 上位ファネル（認知） | 検索/ディスプレイ/動画/ショッピング/デマンドジェネレーション/P-MAX | ニーズベース（興味関心） |
| **購買意向セグメント** | 積極的に商品・サービスを調査中のユーザー | 中〜下位ファネル（検討・購入） | 検索/ディスプレイ/動画/ショッピング/デマンドジェネレーション/P-MAX | 購買行動ベース |
| **詳しいユーザー属性** | 年齢・性別を超えた長期的な属性情報 | 全ファネル | 全キャンペーン（一部は検索のみ） | デモグラフィックベース |
| **ライフイベント** | 人生の大きな転機にいるユーザー（約6ヶ月間） | 中位ファネル | ディスプレイ/動画/デマンドジェネレーション/P-MAX | ライフイベントベース |
| **カスタムセグメント** | キーワード・URL・アプリで独自定義 | 中〜下位ファネル | ディスプレイ/動画/デマンドジェネレーション/P-MAX | 検討行動ベース |
| **自社データセグメント** | 自社サイト訪問者・CRM・アプリユーザー | 下位ファネル（リマーケティング） | 全キャンペーン | 購買行動ベース |
| **類似セグメント** | 自社データに類似するユーザー | 中位ファネル | デマンドジェネレーションのみ | 拡張リーチ |
| **統合セグメント** | 複数セグメントのAND/OR組み合わせ | 任意 | 全キャンペーン | 複合軸 |

## 2. 各セグメントの詳細設定

### A. アフィニティセグメント

長期的な趣味嗜好・ライフスタイルに基づく。約150セグメント、12の親カテゴリ。

**主要カテゴリ一覧：**

| 親カテゴリ | 代表的なセグメント例 | ビジネスモデル型との親和性が高い型 |
|:---------|:----------------|:------------------|
| 銀行・金融 | 投資家、保険愛好者 | B-2、C-1 |
| 美容・健康 | 美容愛好者、健康志向者 | E-2、D-1 |
| 食品・ダイニング | 料理愛好者、グルメ、外食好き | A-1、A-3、D-3 |
| 住まい・庭 | DIY愛好者、インテリア好き | B-1、N-1 |
| ライフスタイル・趣味 | アウトドア派、ファッション好き | D-1、D-2 |
| メディア・エンターテインメント | 映画ファン、ゲーマー、読書家 | A-2、C-2、J-2 |
| ニュース・政治 | ニュース愛好者 | — |
| ショッピング | バーゲンハンター、高級品志向 | I-1、N-1 |
| スポーツ・フィットネス | スポーツファン、フィットネス愛好者 | E-2 |
| テクノロジー | ハイテク愛好者、ソーシャルメディアユーザー | C-1、D-2 |
| 旅行 | 旅行好き（国内/海外） | L-2、F-2 |
| 自動車 | 自動車愛好者 | D-2 |

**設定方法：**
```
管理画面パス：キャンペーン/広告グループ > オーディエンス > オーディエンスセグメントの編集
  > 閲覧 > ユーザーの興味や関心、習慣（アフィニティ）
設定レベル：キャンペーンまたは広告グループ
モード：ターゲティング（配信制限）またはモニタリング（入札調整のみ）
```

### B. 購買意向セグメント（In-Market）

積極的に商品・サービスを調査・比較中のユーザー。約26の親カテゴリ。

**主要カテゴリ一覧：**

| 親カテゴリ | 代表的なサブカテゴリ例 | 想定される活用シーン |
|:---------|:----------------|:----------------|
| アパレル・アクセサリー | 紳士服、婦人服、靴、ジュエリー | D-1型のファッションブランド |
| 自動車 | 新車/中古車（メーカー別）、パーツ | D-2型 |
| ベビー・子供用品 | ベビー服、おもちゃ、チャイルドシート | B-1型のライフイベント連動 |
| 美容・パーソナルケア | スキンケア、ヘアケア、美容機器 | D-1型、A-1型 |
| ビジネスサービス | 会計、HR、マーケティング | B-2型、C-1型 |
| 教育 | 語学、資格、大学院 | E-1型 |
| 金融サービス | 保険、投資、ローン | B-1型 |
| ギフト・特別な機会 | ギフトバスケット、花、パーソナライズギフト | **D-3型に最重要** |
| 家庭・ガーデン | 家具、キッチン、リフォーム | B-1型 |
| 不動産 | 住宅購入/賃貸 | B-1型 |
| 旅行 | 航空券、ホテル、パッケージツアー | L-2型、F-2型 |
| ソフトウェア | ビジネスソフト、セキュリティ | C-1型、J-1型 |

**設定方法：**
```
管理画面パス：キャンペーン/広告グループ > オーディエンス > オーディエンスセグメントの編集
  > 閲覧 > ユーザーが積極的に調べている情報や立てている計画（購買意向）
設定レベル：キャンペーンまたは広告グループ
モード：ターゲティングまたはモニタリング
```

### C. 詳しいユーザー属性（Detailed Demographics）

基本的な年齢・性別・世帯収入を超えた属性情報。

| カテゴリ | 選択肢 | 対応キャンペーン |
|:--------|:------|:-------------|
| **子供の有無** | 乳児の親/幼児の親/未就学児の親/小学生の親/中高生の親 | 全キャンペーン |
| **配偶者の有無** | 独身/交際中/既婚 | 全キャンペーン |
| **学歴** | 現役大学生/高卒/学士/大学院卒 | 全キャンペーン |
| **住居** | 持ち家/賃貸 | 全キャンペーン |
| **雇用** | 企業規模/業種 | **検索キャンペーンのみ** |

**設定方法：**
```
管理画面パス：キャンペーン/広告グループ > オーディエンス > ユーザー属性
  > 詳しいユーザー属性
設定レベル：キャンペーンまたは広告グループ
基本属性（年齢・性別・世帯収入）は広告グループレベルで設定
```

### D. ライフイベント

人生の大きな節目にいるユーザー（前後約6ヶ月間）。

| ライフイベント | タイミング修飾 | ビジネスモデル型との親和性 |
|:------------|:----------|:------------|
| 起業 | まもなく/最近 | B-2、B-3、C-1 |
| 大学卒業 | まもなく/最近 | E-1 |
| 転職 | まもなく/最近 | M-2 |
| 結婚 | まもなく/最近 | B-1、D-3 |
| 引越し | まもなく/最近 | B-1 |
| ペット（新規） | 犬/猫 | A-1 |
| 住宅購入 | まもなく/最近 | B-1 |
| 定年退職 | まもなく/最近 | — |
| 出産・育児 | まもなく/最近 | B-1 |

**設定方法：**
```
管理画面パス：キャンペーン/広告グループ > オーディエンス > オーディエンスセグメントの編集
  > 閲覧 > ユーザーのライフイベント
設定レベル：キャンペーンまたは広告グループ
対応キャンペーン：ディスプレイ/動画/デマンドジェネレーション/P-MAX
※検索・ショッピングキャンペーンでは使用不可
```

### E. カスタムセグメント

自社独自の定義でオーディエンスを作成。3つのパラメータで構成。

| パラメータ | 内容 | 設定のコツ |
|:---------|:----|:---------|
| **キーワード** | 2つのモードあり：①興味関心・購入意向（デフォルト）②Google検索語句 | ②はGoogle検索面のみ有効。具体的な商品名・悩みKWが効果的 |
| **URL** | ターゲットが閲覧しそうなサイトのURL | 指定URLそのものではなく「類似サイト」の訪問者がターゲット |
| **アプリ** | ターゲットが使用しそうなアプリ | 指定アプリそのものではなく「類似アプリ」の利用者がターゲット |

**設定方法：**
```
管理画面パス：ツールと設定 > 共有ライブラリ > オーディエンスマネージャー > カスタムセグメント > ＋
設定項目：
  1. セグメント名
  2. モード選択：「興味関心・購入意向」or「Google検索語句」
  3. キーワード入力（複数可）
  4. URL入力（任意・複数可）
  5. アプリ入力（任意・複数可）
対応キャンペーン：ディスプレイ/動画/デマンドジェネレーション/P-MAX
※検索・ショッピングキャンペーンでは使用不可
```

**セグメント軸別の推奨カスタムセグメント設計：**

| セグメント軸 | キーワード例 | URL例 |
|:----------|:---------|:-----|
| 検討行動（状況） | 「〇〇 比較」「〇〇 おすすめ」「〇〇 口コミ」 | 競合サイト、比較サイト、口コミサイト |
| 購買行動 | 「〇〇 通販」「〇〇 購入」「〇〇 安い」 | ECモール、専門店EC |
| ニーズ | 「〇〇 効果」「〇〇 メリット」「〇〇 始め方」 | 情報メディア、ブログ |
| ライフイベント | 「母の日 ギフト」「引越し 準備」「転職 タイミング」 | ギフトサイト、ライフイベント系メディア |

### F. 自社データセグメント（Your Data / リマーケティング）

| データソース | 設定要件 | 保持期間 | 設定パス |
|:----------|:------|:------|:------|
| **ウェブサイト訪問者** | Googleタグ設置済み | 最大540日 | オーディエンスマネージャー > ＋ > ウェブサイトを訪問したユーザー |
| **アプリユーザー** | Firebase/SDK連携 | 最大540日 | オーディエンスマネージャー > ＋ > アプリユーザー |
| **カスタマーマッチ（CRM）** | CSV（メール/電話/住所） | 最大540日 | オーディエンスマネージャー > ＋ > 顧客リスト |
| **YouTubeユーザー** | YouTubeチャンネル連携 | 最大540日 | オーディエンスマネージャー > ＋ > YouTubeユーザー |

**カスタマーマッチの設定詳細：**
```
ファイル形式：CSV（ASCII or UTF-8）
最小レコード数：100件
必須ヘッダー（英語）："Email", "Phone", "First Name", "Last Name", "Country", "Zip"
ハッシュ化：SHA256（事前ハッシュ or Googleが自動ハッシュ）
正規化ルール：
  - メール：小文字化、前後の空白除去
  - 電話：E.164形式（例：+81XXXXXXXXXX）
  - Gmail/Googlemail：ピリオドとプラス記号以降を除去してからハッシュ
EU要件：EEAデータには同意シグナルが必須
```

**ウェブサイト訪問者のルール設定例：**
```
ルール種別：
  - 特定ページ訪問者（URL条件）
  - 特定イベント（購入、カート追加等）
  - 滞在時間（上位X%）
  - 訪問したが特定ページは未訪問（除外組み合わせ）

設定例（D-3型 ギフトEC）：
  名前：「過去180日以内のサイト訪問者（購入なし）」
  条件：URLに「gyoren.net」を含む AND コンバージョン「purchase」が0件
  期間：180日
```

### G. Performance Max（P-MAX）のオーディエンスシグナル

P-MAXではオーディエンスを「シグナル（推奨）」として設定する。ハードターゲティングではなく、Googleの最適化の起点として使用される。

**シグナルとして使える種類：**

| シグナル種別 | 内容 | 推奨用途 |
|:----------|:----|:------|
| 自社データ | 顧客リスト、サイト訪問者リスト | 最優先シグナル。類似ユーザーの探索起点 |
| カスタムセグメント | キーワード/URL/アプリ | 検討行動シグナル |
| 興味関心と詳しいユーザー属性 | アフィニティ/購買意向/ライフイベント/詳しいユーザー属性 | 興味関心シグナル |
| 検索テーマ | キーワードに近いテーマ入力 | 検索意図シグナル |

**設定方法：**
```
管理画面パス：P-MAXキャンペーン > アセットグループ > オーディエンスシグナル > 編集
設定項目：
  1. 自社データ（セグメントを選択）
  2. カスタムセグメント（新規作成 or 既存選択）
  3. 興味関心と詳しいユーザー属性（セグメント選択）
  4. 検索テーマ（キーワード入力）
注意：シグナルは推奨であり、Googleは設定外にも配信を拡張する
最適化期間：約2週間で統合完了
```

## 3. ターゲティングモード

| モード | 動作 | 推奨用途 |
|:------|:----|:------|
| **ターゲティング** | 選択したオーディエンスにのみ配信 | ディスプレイ/動画のデフォルト。リマーケティング |
| **モニタリング（観察）** | 全ユーザーに配信しつつ、オーディエンスの実績を可視化 | 検索/ショッピングの推奨。入札調整で優先度を付ける |

**入札調整倍率：** 0.1〜10.0（-90%〜+900%）

## 4. STEP4出力テンプレートへの適用ガイド

### セグメント軸 → Google広告設定の変換ルール

| セグメント軸 | 第1候補の設定方法 | 第2候補（第1が使えない場合） |
|:----------|:-------------|:---------------------|
| デモグラフィック（属性） | 基本属性（年齢・性別・収入）＋詳しいユーザー属性 | — |
| 購買行動（行動） | 購買意向セグメント＋自社データ（カスタマーマッチ） | カスタムセグメント（購入系KW） |
| 検討行動（状況） | カスタムセグメント（検索KW・競合URL） | 購買意向セグメント |
| 興味関心（ニーズ） | アフィニティセグメント | カスタムセグメント（興味KW） |
| ライフイベント（時間） | ライフイベントセグメント | カスタムセグメント（ライフイベントKW） |

### 自社データの有無による設定パターン

| 自社データの状況 | 推奨設定パターン |
|:------------|:-----------|
| **CRMリストあり（1,000件以上）** | カスタマーマッチ＋類似セグメント（デマンドジェネレーション）が最優先 |
| **サイトタグあり・訪問者あり** | リマーケティングリスト＋カスタムセグメント |
| **自社データなし** | カスタムセグメント＋購買意向＋アフィニティの組み合わせ |

## 5. 設定時の注意事項

- **検索キャンペーン**ではアフィニティ/購買意向/ライフイベントは「モニタリング」推奨（ターゲティングにすると配信量が激減する）
- **カスタムセグメント**は検索・ショッピングでは使用不可
- **ライフイベント**は検索・ショッピングでは使用不可
- **P-MAX**ではすべてのオーディエンスが「シグナル」扱い（ハードターゲティングではない）
- **類似セグメント**はデマンドジェネレーションキャンペーンでのみ利用可能
- 統合セグメントで「AND条件」を使うとリーチが過度に狭まるリスクがある（最低でも推定リーチ1,000以上を確保）
