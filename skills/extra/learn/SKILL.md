# Session Learn（知識抽出）

## 手順

1. ユーザーとのセッションで確定した学び・手順・落とし穴を整理する
2. パターンを一般化できるか判断する（非自明・再現性・検証済みを優先）
3. スキル名の案をケバブケースで提案する（例: `nextjs-cache-patterns`）
4. ユーザーに仕様・保存先（個人用 `~/.cursor/skills/` かプロジェクトの `.cursor/skills/`）を確認する
5. 合意後、`SKILL.md` のドラフト（フロントマター + 本文）を提示する

## スキル化の判断

- 毎回同じ説明をしている作業ほど候補
- 詳細は `references/` に分け、本文は 5000 語以内を目安にする
