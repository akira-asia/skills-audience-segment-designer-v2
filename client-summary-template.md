# クライアント提示用サマリ テンプレート
# audience-segment-designer v4.1 参照ファイル
# 更新日：2026-03-23

このファイルはSTEP3で参照する。STEP2-5までの分析結果をもとに、以下の①〜⑥構成の1本のmdファイルを生成する。

**重要：** v4.1からSTEP3の出力は1ファイル統合。旧v3.x形式（3点セット）は出力しない。

---

## 出力ファイル仕様

- **ファイル名：** `executive-segment-brief-[クライアント名]-[対象5seg].md`
- **形式：** Markdown（`.md`）
- **対象読者：** 初見の意思決定者が上から読むだけで全体像を把握できること

---

## 必須セクション①〜⑥テンプレート

---

### 冒頭：「このドキュメントの読み方」ブロック

```markdown
# [クライアント名]|広告セグメント設計ブリーフ
クライアント：[正式名称]|作成日：YYYY-MM-DD

---

## このドキュメントの読み方

(コードブロックで記述)
① 「現状と課題」  → 今何が起きているかを把握する
② 「設計の核心」  → 今回の広告戦略の考え方を理解する
③ 「セグメント一覧」 → 誰に・何を・どの順で打つかを確認する
④ 「各セグメント詳細」 → 動く前の根拠と注意点を確認する
⑤ 「施策原則・NGワード」 → 絶対にやってはいけないことを確認する
⑥ 「次のアクション」 → 具体的な作業を開始する
(コードブロック終わり)
```

### ① 現状と課題

**必須要素：**

- 「一言で言うと」ブロック：現状の構造問題を引用符付きで1〜2文に凝縮する
- KPI推移表：数値を必ず含める
- 顧客構造の歪みをASCIIアートで視覚化する（理想形 vs 現状形を並べる）
- 3つの優先課題を表形式で整理する

```markdown
## ① 現状と課題|「[問題の本質を一言で]」

### 一言で言うと

> [現状の構造問題を1〜2文。「このまま放置すると〇〇になるリスクがある」まで含める]

### 数字で見る現状

| 指標 | [過去年] | [現在年] | 診断 |
|------|---------|---------|------|
| 年間売上 | | | |
| 総顧客数 | | | |
| [最重要KPI] | | | [絵文字]問題あり/成長中 |
| [次点KPI] | | | |

### 顧客構造の歪み（5seg分析）

(コードブロックで記述)
【理想の形：台形】           【現状の形：〇〇型】

     ロイヤル                  ロイヤル   〇人  売上〇%  ←[診断]
   ──────────            ──────────────────────────
       一般                    一般      〇人  売上〇%
  ──────────────        ──────────────────────────────
     トライアル                トライアル 〇人  売上〇%  ←[診断]
────────────────────

[BM型の理想像とのギャップを1〜2文で説明]
(コードブロック終わり)

### 3つの優先課題

| 課題 | 現状値 | 改善目標 | 改善インパクト（試算） |
|------|--------|---------|-------------------|
| **🔴課題①** [課題名] | [現状] | [目標] | **年間＋〇〇円** |
| **🟠課題②** [課題名] | [現状] | [目標] | **年間＋〇〇円** |
| **🟠課題③** [課題名] | [現状] | [目標] | **年間＋〇〇円** |
```

**品質チェック：**
- [ ] 定量データ（数値）が含まれているか
- [ ] ASCIIアートによる顧客構造の可視化があるか
- [ ] 改善インパクト試算が含まれているか

### ② 設計の核心

**必須要素：**

- 最も重要な発見を罫線囲みのコードブロックで強調する（定量データ必須）
- As-Is → To-Be のブランド認知変化をコードブロックで図示する

```markdown
## ② 設計の核心|「[設計方針を一言で]」

### 今回の広告設計で最も重要な発見

(コードブロックで記述)
[比較対象A]  →  [数値A]

[比較対象B]  →  [数値B]

               差：[差分]
(コードブロック終わり)

**→ [この発見から導かれる広告設計の方針を1〜2文で]**

### ブランド認知を「書き換える」戦略

(コードブロックで記述)
現在の顧客認識（As-Is）            目指す顧客認識（To-Be）

「[現在の認識]」            →    「[目指す認識]」
(コードブロック終わり)
```

**品質チェック：**
- [ ] コードブロック内で数値が強調されているか
- [ ] As-Is / To-Be が明確に対比されているか

### ④ 各セグメントの詳細

**必須要素：**
- 「この人たちは誰か」→「なぜこの優先度か」→「✅やること / ❌やってはいけないこと」の順で必ず記述する
- 広告対象外セグメントには必ずStep形式の代替手段を記述する

```markdown
## ④ 各セグメントの詳細

---

### S[N]|[セグメント名] [優先度絵文字][優先度]・予算〇%

**この人たちは誰か**

- [ペルソナ・定義を箇条書きで3〜5点]
- [CRMリストの有無・即配信可否も記載]

**なぜ[最優先/高優先]か**

| 理由 | 詳細 |
|------|------|
| [理由1] | [根拠・数値] |
| [理由2] | [根拠・数値] |
| [理由3（インパクト試算）] | [年間+〇〇円] |

**広告の打ち方**

(コードブロックで記述)
✅ やること
「[クリエイティブアングル名]」クリエイティブ
例：「[具体的なコピー例]」
→ [なぜこのアプローチが有効かを1文で]

❌ やってはいけないこと
[NG施策名]
→ [なぜNGかを1文で]
(コードブロック終わり)

---

### S[N]|[広告対象外セグメント名] ─広告対象外

**今すぐ広告を出さない理由**

- [理由1]
- [理由2]

**代わりにやること（MA・コンテンツ先行）**

(コードブロックで記述)

Step 3：[具体的なアクション]
(コードブロック終わり)
```

**品質チェック：**
- [ ] 全セグメントに「この人たちは誰か」があるか
- [ ] 全最優先・高優先セグメントに「なぜ優先か」の表があるか
- [ ] 全セグメントに「✅やること / ❌やってはいけないこと」のコードブロックがあるか
- [ ] 広告対象外セグメントにStep形式の代替手段があるか

### ⑤ 施策原則・NGワード

**必須要素：**

- NG番号付きの表形式で出力する
- 「禁止」だけでなく「なぜNGか」を必ず添える
- BM型の禁忌（`bm-segment-axis-map.md`の「型の禁忌」列）を必ず反映する

```markdown
## ⑤ 施策原則・NGワード

### 全媒体・全セグメント共通のNGルール

| # | やってはいけないこと | 理由 |
|---|-------------------|------|
| **NG①** | [NGワード・施策] | [ブランド・数値・構造的理由を含める] |
| **NG②** | [NGワード・施策] | [なぜNGか] |
| **NG③** | [NGワード・施策] | [なぜNGか] |
| **NG④** | [NGワード・施策] | [なぜNGか] |
```

**品質チェック：**
- [ ] 各NGワードに「なぜNGか」の理由が添えられているか（「禁止」だけは不可）
- [ ] BM型禁忌が反映されているか

### ⑥ 次のアクション

**必須要素：**

- 「今週中」「来週中」「配信開始から〇週間後」の時系列でコードブロックに整理する
- 各アクションは「〇〇を抽出する」「〇〇を設定する」等の具体動詞で始める（「検討する」禁止）
- 末尾にExcelの参照先（シート名付き）を案内する

```markdown
## ⑥ 次のアクション|この順番で動く

(コードブロックで記述)
【今週中】
─────────────────────────────────────────────
□ Step 1：[具体動詞で始まるアクション]
         ├ [サブアクション]
         └ [サブアクション]
□ Step 2：[具体動詞で始まるアクション]
□ Step 3：[具体動詞で始まるアクション]

【来週中】
─────────────────────────────────────────────
□ Step 4：[具体動詞で始まるアクション]
□ Step 5：[具体動詞で始まるアクション]

【配信開始から2週間後】
─────────────────────────────────────────────
□ Step 6：[CTR・CVRを比較してアングルと予算配分を見直す]
□ Step 7：[将来セグメントの参入可否を判断する]
(コードブロック終わり)

---

**詳細なターゲティング設定はExcelを参照**

(コードブロック終わり)
```

**品質チェック：**
- [ ] アクションが「今週中 / 来週中 / 〇週間後」の時系列コードブロックになっているか
- [ ] 各Stepが具体動詞で始まっているか（「検討する」は禁止）
- [ ] 末尾にExcelの参照先（シート名付き）が案内されているか

## NG出力パターン（絶対に出力しない）

以下の形式で出力した場合はSTEP3をやり直す。

- `client-summary-[クライアント名]-[対象5seg]-executive.md`（旧形式・廃止）
- `client-summary-[クライアント名]-[対象5seg]-segments-priority.md`（旧形式・廃止）
- `client-summary-[クライアント名]-[対象5seg]-segments-approach.md`（旧形式・廃止）
- 「なぜこの定義か」「なぜこの優先度か」しか書かれていない箇条書き形式（「✅❌」がない）
- 次のアクションが「〇〇を検討する」「〇〇を実施する方向で」等の曖昧な表現で終わるもの
