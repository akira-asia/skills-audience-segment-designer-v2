# 5seg別・セグメント設計ロジック
# audience-segment-designer v3.2 参照ファイル
# 更新日：2026-03-17

このファイルはSTEP2-1で参照する。入力【4】の「対象セグメント」が確定したら、該当セクションの推奨軸・失敗パターンを読み取りSTEP2-1の設計に反映する。

---

## 前提：5segごとに「持っているデータ」と「変えたい行動」が違う

| 対象5seg | 持っているデータ | 変えたい行動 | セグメント設計の目的 |
|---------|--------------|------------|------------------|
| ロイヤル | 購買履歴・CRM・行動ログ | 維持 → さらなるLTV拡大 | 「崩れそうな人」と「さらに育てられる人」を見分ける |
| 一般 | 購買履歴・メール開封等 | 一般 → ロイヤル化 | 「何が引き金でロイヤル化するか」で分ける |
| トライアル | 初回購買データ・流入経路 | トライアル → F2転換 | 「F2に至る確率が高い人」と「離脱しそうな人」を分ける |
| 未顧客 | なし（広告・市場データのみ） | 未顧客 → 初回購買 | 「なぜ買っていないか」の理由で分ける |
| 離脱 | 過去の購買履歴・離脱時期 | 離脱 → 復帰 | 「なぜ離脱したか」の原因で分ける |

---

## ① ロイヤル：「崩れる予兆」と「さらに伸びる余地」で分ける

### 推奨セグメント軸

```
軸A：購買頻度の安定度（安定ロイヤル vs 頻度が微減しているロイヤル）
軸B：カテゴリ深度（1カテゴリ集中 vs 複数カテゴリ横断）
軸C：紹介・口コミ行動（発信している vs していない）
```

### セグメント × 施策マッピング

| セグメント | 特徴 | 打ち手 |
|----------|-----|--------|
| 安定×多カテゴリ×発信あり | 真のロイヤル。コアファン | アンバサダー化・UGC活用 |
| 安定×1カテゴリ集中 | カテゴリロイヤル。LTV上限がある | クロスセル訴求 |
| 頻度微減×発信なし | 離脱の予兆。最優先で手を打つ | 理由調査＋個別オファー |

### 避けるべき失敗

全員に同じ「ありがとう」施策を当てること。ロイヤルは均質に見えて、「真のファン」と「崩れかけ」が混在している。後者を放置するとロイヤルの総数が静かに減少し続ける。

---

## ② 一般：「何がロイヤル化の引き金か」で分ける

### 推奨セグメント軸

```
軸A：購入動機（機能目的 vs 感情・共感 vs 価格）
軸B：購買文脈（定期的な用途あり vs 思い出し買い）
軸C：情報接触（ブランドコンテンツに触れているか）
```

### セグメント × 施策マッピング

| セグメント | 特徴 | 打ち手 |
|----------|-----|--------|
| 感情共感×定期用途あり | ロイヤル化一歩手前。体験の「意味付け」が効く | ブランドストーリー・会員特典 |
| 機能目的×思い出し買い | 競合スイッチリスク高。価格で来ている | 利便性訴求＋サブスク・定期便誘導 |
| 価格動機×不定期 | 施策コスト対効果が低い可能性 | 除外候補として評価する |

### 避けるべき失敗

全員にロイヤル化施策を当てること。価格動機で来た一般顧客にブランド体験訴求をしても響かない。まず購入動機で分けることが設計の起点になる。

---

## ③ トライアル：「F2転換の確率」と「離脱の理由」で分ける

### 推奨セグメント軸

```
軸A：流入経路（指名検索 vs 広告 vs 紹介 vs 偶然）
軸B：購入文脈（ギフト vs 自家消費 vs 試し買い）
軸C：初回購買後の行動（サイト再訪・メール開封・レビュー投稿）
```

### セグメント × 施策マッピング

| セグメント | 特徴 | 打ち手 |
|----------|-----|--------|
| 指名検索×自家消費×再訪あり | F2確率が高い。早期に背中を押す | 購入後7日以内のステップメール・次回クーポン |
| 広告経由×ギフト購入 | 本人のリピート意向が低い可能性。ギフト受取人→顧客化の可能性あり | ギフト受取人へのアプローチ設計（D-3型の典型施策） |
| 偶然購入×初回後行動なし | 離脱確率高。過度な投資は非効率 | 低コストのリマインドのみ |

### 避けるべき失敗

F2転換確率を無視して全員に均等投資すること。流入経路と購入文脈でF2確率は大きく変わる。広告経由のギフト購入者と指名検索の自家消費者に同じ施策を当てると費用対効果が大幅に落ちる。

---

## ④ 未顧客：「なぜ買っていないか」の理由で分ける

※ 5segの中で最も軸の幅出しが重要なセグメント。データがない分、市場・競合・心理的障壁からの推定が中心になる。

### 推奨セグメント軸

```
軸A：認知状況（知らない vs 知っているが未検討 vs 検討して離脱）
軸B：購入しない理由（カテゴリ障壁 vs 競合使用中 vs 価格障壁 vs 必要性未認知）
軸C：購入動機の種類（状況ベース：どんな状況で買いそうか）
```

### セグメント × 施策マッピング

| セグメント | 特徴 | 打ち手 |
|----------|-----|--------|
| 認知なし | カテゴリ自体を知らない | 認知拡大・カテゴリ教育型クリエイティブ |
| 認知あり×競合使用中 | スイッチャー候補。最もROIが高いことが多い | 比較訴求・スイッチ障壁の除去 |
| 認知あり×カテゴリ障壁 | 「自分には関係ない」と思っている | 障壁破壊型アングル（「失敗しない」「簡単」等） |
| 検討済み×離脱 | 一度は興味があった。最も復帰しやすい | リターゲティング＋離脱理由の仮説検証 |

### 避けるべき失敗

認知段階が違う人に同じクリエイティブを当てること。「知らない人」と「検討して離脱した人」では必要なメッセージが正反対になる。前者にはカテゴリ教育、後者には「あの時の懸念を解消する」訴求が効く。

## ⑤ 離脱：「なぜ離脱したか」の原因で分ける

### 推奨セグメント軸

```
軸A：離脱理由（忘却型 vs 不満型 vs 目的達成型 vs ライフステージ変化型）
軸B：離脱からの経過時間（直近6ヶ月 vs 1年以内 vs 1年以上）
軸C：離脱前のロイヤリティ（ロイヤルからの離脱 vs 一般からの離脱）
```

### セグメント × 施策マッピング

| セグメント | 特徴 | 打ち手 |
|----------|-----|--------|
| 忘却型×直近 | 不満なく惰性で離れた。最も復帰確率が高い | リマインド型・「お久しぶり」クリエイティブ |
| 不満型（品質・CS） | 同じ訴求では戻らない。改善が前提 | 改善報告型クリエイティブ＋個別謝罪 |
| 目的達成型 | E型（資格・コーチング）に多い。卒業として設計すべき | 次のステップ提案（上位サービスへのアップセル） |
| ロイヤルからの離脱 | 最優先で原因究明。放置すると口コミリスクあり | 1on1アンケート・個別コンタクト |

### 避けるべき失敗

原因を問わず全員にクーポンを送ること。特に不満型の離脱者に価格訴求をしても、「また同じ体験をしてお金を損する」とネガティブに受け取られる可能性が高い。離脱理由の分類が施策選択の大前提になる。

---

## まとめ：5seg × セグメント軸の選び方マトリクス

| 対象5seg | 主軸 | データ源泉 | 最も避けるべき失敗 |
|---------|-----|----------|-----------------|
| ロイヤル | 行動ベース（頻度変化・カテゴリ幅） | CRM・購買ログ | 全員に同じ「ありがとう」施策 |
| 一般 | ニーズ・購入動機ベース | 購買文脈・コンテンツ接触 | 全員にロイヤル化施策を当てる |
| トライアル | 流入経路×購入文脈ベース | 流入ログ・初回後行動 | F2確率を無視して全員均等投資 |
| 未顧客 | 認知状況×購入しない理由ベース | 広告データ・競合分析 | 認知段階が違う人に同じクリエイティブ |
| 離脱 | 離脱理由×経過時間ベース | 購買履歴・解約理由 | 全員にクーポン訴求（不満型に特に逆効果） |

## STEP2-1への適用方法

1. 入力【4】の「対象セグメント」を確認する
2. 該当セクションの「推奨セグメント軸」を読み取る
3. BM型の推奨軸（STEP2-0 / `bm-segment-axis-map.md`）と組み合わせ、最終的な軸の優先順位を決定する
4. 「避けるべき失敗」を確認し、STEP2-5のNGワード・設計留意点に反映する

### 軸の組み合わせ方（優先順位）

```
第1優先：5segロジックの主軸（何で消費行動が最も変わるか）
第2優先：BM型の推奨軸（型のボトルネック起点）
第3優先：広告ターゲティング実現性（管理画面で定義できるか）
```

3つが重なるセグメント軸が「最も戦略的根拠のある軸」になる。
